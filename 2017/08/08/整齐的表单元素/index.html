表单对齐排版及表单元素验证

表单元素的整齐排列关乎页面效果，js验证合法性可省去后台验证，提高效率，在开发web应用的过程中，用户可以输入的大部分内容都是在表单的元素中完成的，与后台的交互大多数时候也是通过表单中的按钮来完成的，js最初任务包括分担服务器处理表单的责任，打破处处依赖服务器的局面

HTML

HTML的设计至关重要，HTML设计合理，CSS才能方便实现效果

考虑到label的for属性可以规定label与哪个表单元素绑定，为了方便格式设计，下文示例使用label的隐式绑定，为了设计样式，所有内容都用单独标签包裹

<fieldset> 元素组合表单中的相关数据

<legend> 元素为 <fieldset>元素定义标题。

CSS

在CSS中label的display属性设置成block,规定span标签与input
标签的宽度，设置span标签右对齐

js

在HTML中，表单由<form>元素表示，在js中表单对应HTMLFormElement类型。
常用的属性与方法如下：
action:接受请求的URL;
elements:表单中所有控件集合;
length:表单中控件数量；
method:要发送的http请求类型，通常为“get”或“post”;
name:表单的名称;
reset()：将所有表单域重置为默认值，会像单击按钮一样触发reset事件
submit()：提交表单,HTML中无需包含表单提交按钮，但是不会触发submit 事件，因此要记得在调用此方法之前先验证表单数据。
target:用于发送请求和接收响应的窗口名称;

js获取表单的方式如下：
1.getElementById()
2.document.forms
document.forms可获取所有表单，通过索引或name值取得特定表单
2.1document.forms[0]
2.2document.forms["form2"]
注意：
1.表单中有提交按钮，js提交表单的事件处理程序应添加到submit事件上，而非click，因为有些浏览器会先触发click，再触发submit,若在点击事件中禁用按钮，则表单不会提交。
2.在第一次提交表单后，如果长时间没有反应，用户可能会变得不耐烦。这时候，他们也许会反复单击提交按钮。结果往往很麻烦，解决办法如下：在第一次提交表单后就禁用提交按钮，或者利用onsubmit 事件处理程序取消后续的表单提交操作。

验证

HTML5表单验证

HTML5可验证文本框中内容是不是type对应地址格式的文字，若不是，则不允许提交，常用类型有url,email,date,time,datetime,tel,number,range,search,color
1.type=”email”提交时可为空，除非加上required属性
2.type=”color”提供一个颜色选择器
input标签中的pattern 属性规定用于验证输入字段的模式。
模式指正则表达式

取消表单验证的两种方法：
1.form元素或者submit元素的novalidate属性可以关闭整个表单验证，当整个表单的第二部分需要验证的元素比较多时，可先设置为TRUE，提交一部分，之后再设置为FALSE，打开表单验证，提交第二部分内容
2.利用input元素的formnovalidate属性，当整个表单的第二部分需要验证的元素比较少时，可让验证对单个input表单元素失效

js验证（正则表达式）

new RegExp(“^[a-z\d]+[\w\-\.]@([a-z\d]+[a-z\d\-].)+[a-z]{2,4}$”, “i”);
等价于//,前者注意添加转义字符\
1.//.test(value) 返回TRUE或者FALSE
2.value.match(//)

以下方法浏览器支持度不高，Opera支持

表单元素有checkValidity()方法，返回TRUE或者FALSE，type为email的元素会自动验证是否为有效email地址
表单元素有setCustomValidity(“密码不一致”)方法，用来自定义错误提示信息
