第一章 babel环境配置

babel默认转换句法，而不转换新的API，如Iterator、Generator、set、maps、Proxy、Reflect、Symbol、Promise等全局对象，以及一些定义在全局对象上的方法

let与const

let

块级作用域：在循环体内，设置循环变量的是父作用域，循环体内部是一个子作用域，两者有各自单独的作用域
不存在变量提升：声明的变量必须在声明后使用，否则报错（注：var声明但不赋值，let不声明，不赋值）
暂时性死区：区块对声明的变量形成封闭作用域，外部作用域对该变量的声明对其无影响，区块内变量声明完成前都是该变量的死区，
typeof未声明的变量不会报错，typeof位于死区内变量会报错
不允许重复声明：一个作用域只能声明一次
为什么需要块级作用域？
外层作用域与内层相互独立，外层作用域无法读取内层变量，内层可读取外层变量，内层作用域可定义外层作用域同名变量
块级作用域与函数声明：块级作用域内可以声明函数（必须在{}内，否则报错），类似于let声明，因此，在块级作用域外不可引用该函数，但是ES6浏览器仍看做var声明
function f() { console.log('I am outside!'); } (function () { if (false) { // 重复声明一次函数f function f() { console.log('I am inside!'); } } f(); }());
输出：报错
function f() { console.log('I am outside!'); } (function () { if (true) { // 重复声明一次函数f function f() { console.log('I am inside!'); } } f(); }());
输出：I am inside!
function f() { console.log('I am outside!'); } (function () { f(); }());
输出: I am outside!
结论：但是ES6浏览器仍把块内函数声明看做var声明，变量提升，因此最好使用函数表达式代替声明函数

do表达式
由于块级作用域内部是多个语句，且外部无法访问内部值，因此使用do表达式让作用域返回值，示例：
let x = do{ let t = 7; t*t+1; }
注意：这只是一个提案

const

const是一个常量，一旦声明就立即初始化，且不能改变，改变常量值会报错
与let一样，const只在块级作用域内有效，不存在变量提升，存在暂时性死区，不可重复声明
本质：const保证的不是值不能改动，而是地址不能改动，对于简单类型数据，值保存在变量指向的那个地址，等同于常量，对于复合类型数据（数组与对象），变量指向的内存地址，保存的只是一个指针，只保证指针是固定的，至于指向的数据结构是否发生变化无法控制
const foo = {}; foo.prop = 123; foo.prop // 123 foo=3//TypeError
const声明对象，对该对象添加属性可以成功，但不能对该对象赋值
若要求不能对对象添加删除属性，则需要将对象冻结const foo=Object.freeze({})，但仍可对已经存在的属性做出修改
将对象彻底冻结
var constantize = (obj) => { Object.freeze(obj); Object.keys(obj).forEach( (key, i) => { if ( typeof obj[key] === 'object' ) { constantize( obj[key] ); } }); };
es6声明对象6种方法，class import let const

顶层对象的属性：
window 浏览器环境下顶层对象
global node环境下顶层对象
es5中顶层对象属性与全局变量是等价的
es6开始顶层对象将逐渐与全局对象脱钩
let const class 声明的全局变量，不属于顶层对象的属性，为保持兼容性，var function依然属于
为了能在各种环境取到顶层对象，一般使用this，this也有一定局限性，如下：全局环境，this返回顶层对象，es6模块与node模块中，this返回当前模块
获取顶层对象的方法：
var getGlobal = function () { if (typeof self !== 'undefined') { return self; } if (typeof window !== 'undefined') { return window; } if (typeof global !== 'undefined') { return global; } throw new Error('unable to locate global object'); };

3.解构赋值

变量的解构赋值

es6按照一定模式，从数组和对象中提取值，对变量进行赋值，称为解构(Destructuring)
let [a,b,c] = [1,2,3]//从数组中提取值，按照对应位置，对变量赋值，本质属于“模式匹配”
右边值少于左边值，解构不成功，变量值等于undefined，右边值对于左边值，解构可以成功
只有数组转为对象以后具备Iterator，所以等号右边也必须是数组,只要某种数据结构具备Iterator，就可以采用数组方式解构赋值
对于Set结构也可以用数组的解构赋值
let [a,b,c] = new Set(['a','b','c'])

解构赋值允许指定默认值 let [a,b=3,c] = [1,2,3]
只有赋予变量的值严格等于undefined，才会使用默认值，null不严格等于undefined

对象的解构赋值

let {foo,bar} = {foo:"123",bar:67} 对象的属性次序是随意的，取值时变量名与属性名同名才能取到值
若不一致还想取到值，可写成let {bar:foo} = {bax:"123",bar:67} foo//67 前者是匹配的模式，后者才是变量，真正被赋值的是后者
若相对前者也赋值，可写成let {bar,bar:foo} = {bax:"123",bar:67} :前面都是模式而不是变量
对象的解构也可以使用默认值var {x: y = 3} = {x: 5};只有赋予变量的值严格等于undefined，才会使用默认值
解构不成功，变量值等于undefined
若提前声明，则在解构赋值时会把大括号看成代码块，因此要加上小括号let x;({x} = {x: 1});//必须加上括号，否则会把大括号当做代码块
数组本质是特殊的对象，可以将数组用于特殊的解构赋值示例： let arr = [1, 2, 3];let {0 : first, [arr.length - 1] : last} = arr;

字符串的解构赋值

let [a,b,c] = 'hello' a//h b//e
let {length:len} = 'hello' len//5

数组和布尔值的解构赋值

let {toString: s} = 123;
let {toString: s} = true;
解构赋值规则: 只要等号右边不是对象或数组，就先转换成对象，undefined和null无法转换成对象，所有等号右边=undefined或null会报错

函数参数的解构赋值

[[1, 2], [3, 4]].map(([a, b]) => a + b); // [ 3, 7 ]
区分：
function move({x = 0, y = 0} = {}) { return [x, y]; }
上文是对x,y指定默认值，move({})使用默认值，结果都为0
function move({x, y} = { x: 0, y: 0 }) { return [x, y]; }
上文是对move的参数使用默认值 move({})参数不为空，使用{},x,y结果都为undefined
[1, undefined, 3].map((x = 'yes') => x); undefined会触发函数参数默认值

圆括号问题

模式中最好不要出现圆括号
非声明语句的非模式部分可以使用圆括号[(b)] = [3]; // 正确 ({ p: (d) } = {}); // 正确

解构赋值的用途

1.交换变量的值 [x,y] = [y,x]
2.从函数返回多个值
3.函数参数的定义
4.提取json数据（本质：对象解构赋值）示例：
let jsonData = { id: 42, status: "OK", data: [867, 5309] }; let { id, status, data: number } = jsonData;
5.函数参数默认值
6.遍历map结构
let m = new Map(); m.set('first','hello'); m.set('second','liming'); for(let [key,value] of m){ console.log(key,value); }
7.输入模块的指定方法
const { SourceMapConsumer, SourceNode } = require("source-map");

4.字符串的扩展

codePointAt

codePointAt用来测试一个字符是由两个字节组成还是由四个字节组成
使用for...of循环，因为它会正确识别32位的UTF-16字符,
function is32Bit(c) { return c.codePointAt(0) > 0xFFFF; } is32Bit("𠮷") // true is32Bit("a") // false
默认为十进制，转化为16进制：ch.codePointAt(0).toString(16)

string.fromCodePoint()

作用上恰好与codePointAt相反
String.fromCodePoint(0x20BB7)// “𠮷”
如果该方法有多个参数，则被合并成一个字符串返回
注意:fromCodePoint方法定义在String对象上，而codePointAt方法定义在字符串的实例对象上。

字符串遍历接口

es6为字符串添加了遍历接口，可使用for···of循环遍历，这个遍历接口可以识别大于0xFFFF的码点

at() 替代 charAt()

require('babel-polyfill')//垫片库 console.log('𠮷'.at(0))

区分at与codePointAt at返回具体值’𠮷’，codePointAt返回十进制134071

normalize()

Unicode正规化,可以识别合成的字符，不能识别三个及其以上合成的字符 NFC标准等价合成 NFD标准等价分解
'\u004F\u030C'.normalize('NFC').length // 1 '\u004F\u030C'.normalize('NFD').length // 2

indexOf includes startsWith endsWith

都返回布尔值
includes startsWith endsWith都支持第二个参数，指定开始搜素位置，endsWith与前两个不同，它针对前n个字符
var s = 'Hello world!'; s.startsWith('world', 6) // true s.endsWith('Hello', 5) // true s.includes('Hello', 6) // false

repeat

返回一个新字符串,参数如果是小数，会向下取整，负数或infinite会报错,0-(-1),NAN会默认为0，如果是字符串会转化成数字
"x".repeat(3) //xxx

padStart padEnd

这是一个提案，先加垫片
require('babel-polyfill') console.log('x'.padStart(5, 'ab') ) //ababx
第一个参数用来指定字符串最小长度，第二个用来补全的字符串，若原字符大于最小长度，则返回原字符串
常见用途'09-12'.padStart(10, 'YYYY-MM-DD')

模板字符串``

模板字符串可嵌入变量 ${} {}内部可以放入任意js表达式，可用来运算，以及引用对象属性，还能调用函数，但不能使用未声明的变量
如果{}中最终结果不是字符串，要转化成字符串，{}中放入未声明字符串会报错
模板字符串中若表示多行字符串，可保留空格和缩进，可用trim()来消除

标签模板

String.row()

模板字符串的处理函数，对应于替换变量后的模板字符串

注意：如果是字符串以及其他类型，直接返回FALSE Number.isInteger("15") // false

是一个极小的常量，目的在于为浮点数之间运算设置一个误差范围

Math.trunc()

Math.trunc()用于去除一个数的小数部分，返回整数部分

Math.clz32(0) // 32 Math.clz32(1) // 31 Math.clz32(1 << 1) // 30 Math.clz32(1 << 2) // 29 Math.clz32(1 << 29) // 2

Math.imul()

Math.imul(-2,1) //2
返回两个32位带符号整数相乘的结果,对于数值特别大超过js设置的精度的使用此方法才能返回正确的值

7.函数的扩展

函数参数的默认值

function foo(x=1,y=2){}
参数变量是默认声明的，不需要使用let或者const再次声明
参数默认值是惰性求职，会把参数值直接代入函数体的参数中，不得不运算时才运算求解
设有默认值的参数一般为尾参数

函数的length属性

返回没有默认值的参数个数,rest参数不会计入这个属性，只计入设置了默认值之前的参数
(function (a) {}).length // 1
(function(...args) {}).length // 0
(function (a, b = 1, c) {}).length // 1
let x = 1; function f(y = x) { let x = 2; console.log(y); } f() // 1

设置某参数值不能省略

function throwIfMissing() { throw new Error('Missing parameter'); } function foo(mustBeProvided = throwIfMissing()) { return mustBeProvided; }

rest参数之后，不能再有其他参数
name属性，函数的函数名function foo() {}; foo.name // "foo"
(new Function).name // "anonymous"
bind返回的函数，name会加上bound前缀
function foo()(); foo.bind({}).name //bound foo

箭头函数

如果箭头函数代码块包含语句，需要用大括号括起来，如果返回对象，要用小括号把大括号括起来
注意：函数体内this对象就是定义生效时所在的对象，而不是使用时所在的对象，且this的指向固定不可变
不能做构造函数，与new一起使用会报错
不可以使用arguments
不能使用yield，因此不能使用Generator

函数绑定到对象上

运算符:: 对象名::函数名
如果左边为空，右边是一个对象的方法,等于将该方法绑定到该对象上 对象的方法.bind(对象名)

尾调用优化

ES6的尾调用优化只在严格模式下开启
某个函数的最后一步是调用另一个函数
尾调用必须是最后一步操作
只保留内层函数的调用帧是尾调用优化，节省内存

尾递归

尾调用自身成为尾递归
function tco(f) { var value; var active = false; var accumulated = []; return function accumulator() { accumulated.push(arguments); if (!active) { active = true; while (accumulated.length) { value = f.apply(this, accumulated.shift()); } active = false; return value; } }; } var sum = tco(function(x, y) { if (y > 0) { return sum(x + 1, y - 1) } else { return x } }); sum(1, 100000) // 100001

数组的扩展

扩展运算符…

主要用在函数参数中 ...[1,2,3] = 1,2,3
扩展运算符可以展开数组，类似方法有f.apply(null,args)
注意：扩展运算符用在函数的参数中以及用在数组中参数赋值，只能放在最后一位，否则报错
扩展运算符可以展开字符串，还能识别32位Unicode字符
任何Iterator的接口的对象，都可以用扩展运算符转化为真正的数组，如document.querySelectorAll('div')
字符串，Map结构，Set结构，Generator结构，都带有Iterator的接口

Array.from()

可以将array-like（本质有length属性） 以及 具有Iterator接口对象转化为真正的数组
常见的类似数组的对象是DOM操作返回的NodeList集合，以及函数内部的arguments对象。Array.from都可以将它们转为真正的数组。
Array.from({ length: 3 }); // [ undefined, undefined, undefined ]
Array.from还可以接受第二个参数，作用类似于数组的map方法，用来对每个元素进行处理，将处理后的值放入返回的数组
Array.from(arrayLike, x => x * x); // 等同于 Array.from(arrayLike).map(x => x * x); Array.from([1, 2, 3], (x) => x * x) // [1, 4, 9]

Array.of()

Array.of(3, 11, 8) // [3,11,8] 总是返回参数值组成的数组，如果没有参数值，返回空数组
替代方法:
function ArrayOf(){ return [].slice.call(arguments); }

copyWithin

Array.prototype.copyWithin(target, start = 0, end = this.length)
target为开始替换的位置，start为开始读取位置，end为结束读取位置
[1,2,3,4].copyWithin(1,2) 从2开始替换，替换结果为[1,3,4,4]

find()和findIndex()

find()返回第一个符合条件的成员的值，findIndex()返回第一个符合条件的成员的位置，这两个函数的第一个参数都是一个回调函数
（回调函数接受三个参数，依次为当前值，当前位置，原数组），所有数组成员依次执行此回调函数，没有符合条件的分别返回
undefined，-1，这两个函数的第二个参数用来绑定回调函数的this对象，可省略
[NaN].indexOf(NaN) // -1 [NaN].findIndex(y => Object.is(NaN, y)) // 0

数组的fill方法

fill使用给定值填充一个数组，fill方法还可以接受第二个和第三个参数，用于指定填充的起始位置和结束位置。
['as','s','d'].fill(1,0,1) //[1,'s','d']

数组的entries(),keys(),values()(不支持？)

都返回一个遍历器对象，可以用for…of循环进行遍历，也可以手动调用遍历器对象的next()
[1,2,3].entries().next().value //[0,1]返回的是键值对

数字的includes

返回布尔值,第一个参数表示要搜索的值，第二个参数表示搜索的起始位置，可为负数,从此位置至数组结尾这个范围内搜索
[1,2,NaN].includes(NaN) //true
[1,2,NaN].includes(NaN，-2) //true
[NaN].includes(NaN) // true
[NaN].indexOf(NaN) // -1
区分Map Set的has方法，map的has方法用来查找键名 Set的has方法用来查找键值

数组的空位

空位是没有任何值的，undefined依然是有值的，所以两者不同
0 in [undefined, undefined, undefined] // true 0 in [, , ,] // false
0指的是0号位置，有值返回TRUE，否则返回FALSE
ES5很多方法跳过空位es6将空位转化成undefined，扩展运算符（…）、Array.from（）转空位为undefined，fill（）、copyWithin（）会拷贝空位，for…of循环也会遍历空位
如果改成map方法遍历，空位是会跳过的。entries()、keys()、values()、find()和findIndex()会将空位处理成undefined。

9.对象的扩展

属性的简洁表示法

简洁写法的属性名总是字符串,下文示例中class是字符串，所以可以使用
`var name=”liming”
var person={
name,
class(){},

* test(){},
getName(){
    return this.name;
}

}
console.log(person.getName()) //liming`

方法有name属性 person.getName.name //getName 如果使用了getter、setter,返回的方法名前加上get或者set
const obj = { get foo() {}, set foo(x) {} }; obj.foo.name // TypeError: Cannot read property 'name' of undefined const descriptor = Object.getOwnPropertyDescriptor(obj, 'foo'); descriptor.get.name // "get foo" descriptor.set.name // "set foo"
如果对象方法是一个Symbol值，返回Symbol的描述

Object.is()

es5中==会自动转换数据类型，===则NaN不等于自身且—0全等于+0，所以es6引入Object.is，纠正了===的两个错误

Object.assign()

Object.assign()用于对象的合并,将源对象的所有可枚举属性（不包括继承属性，不可枚举属性）复制到目标对象上，
var target = { a: 1 }; var source1 = { b: 2 }; var source2 = { c: 3 }; Object.assign(target, source1, source2); target // {a:1, b:2, c:3}
若参数不是对象则将其转为对象，null和undefined无法转为对象，在目标对象位置会报错，在源对象位置会跳过,布尔值数字在源对象位置也会被跳过，字符串会被拆分复制
Symbol类型的值会被拷贝
Object.assign是浅拷贝，如果源对象某个可枚举属性值为对象，则拷贝过去的只是引用，只会拷贝属性的值，无法拷贝取值方法与赋值方法，即无法拷贝get set
遇到同名属性，会替换Object.assign([1, 2, 3], [4, 5]) // [4, 5, 3]

属性的可枚举性和遍历

let obj = { foo: 123 }; Object.getOwnPropertyDescriptor(obj, 'foo') // { // value: 123, // writable: true, // enumerable: true, // configurable: true // }
for…in 忽略不可枚举，但会返回继承来的
JSON.stringify() 忽略不可枚举
Object.assign() 忽略不可枚举
Object.keys() 忽略不可枚举
length toString 都是不可枚举属性，以上四种方法都不会遍历到
class的原型的方法都不可枚举
Object.getOwnPropertyDescriptor(class {foo() {}}.prototype, 'foo').enumerable // false
Object.getOwnPropertyDescriptors() 返回指定对象所有自身属性（非继承属性）的描述对象
Object.setPrototypeOf(目标对象，原型对象) 设置原型对象
Object.getPrototypeOf(对象) 获取原型对象

对象的entries(),keys(),values()

遍历的是可枚举的，不遍历继承的和不可枚举的，会过滤名为Symbol的属性，这三个方法供for…of使用
Object.entries方法的另一个用处是，将对象转为真正的Map结构 let map = new Map(Object.entries(obj))

对象的扩展运算符

双重解构赋值必须是最后一个对象，将剩余对象全部放到该对象里，解构赋值是浅拷贝，不会拷贝继承自原型对象的属性，对于属性值为对象或数组的，拷贝的是引用
let { x, y, ...z } = { x: 1, y: 2, a: 3, b: 4 };
var o = Object.create({ x: 1, y: 2 }); o.z = 3; let { x, ...{ y, z } } = o; x // 1 y // undefined z // 3
上面代码中，变量x是单纯的解构赋值，所以可以读取对象o继承的属性；变量y和z是双重解构赋值，只能读取对象o自身的属性，所以只有变量z可以赋值成功。

null传导运算符

// 如果 a 是 null 或 undefined, 返回 undefined // 否则返回 a.b.c().d a?.b.c().d

Symbol

为了保证属性名的唯一性而引入的
Symbol是一种新的数据类型，除此之外还有null，undefined，Array，Object，Number，String，Boolean
let s = Symbol(); typeof s //symbol
不能使用new来生成,因为它不能做对象，也就没有属性
Symbol()可以接受字符串作为参数，表示描述，如果Symbol()的参数是一个对象，就会调用该对象的toString()方法，将其转为字符串，再生成一个Symbol值
有toString方法，s.toString()可以显式转为字符串，Boolean(s) //true !s //false不能转为数字
参数相同的两个Symbol并不全等
Symbol值不能与其他类型值运算
Symbol主要作为属性名，但是不能用点运算符，必须用方括号，否则表示字符串，而不是Symbol ，该属性是公开属性

消除魔术字符串

常用方法，把经常出现的变量写成对象的属性，而属性值等于哪个并不重要，只要不冲突即可

Symbol属性名的遍历

for…in,for…of,Object.keys(),JSON.stringify(),Object.getOwnPropertyNames()都不能遍历到Symbol
Object.getOwnPropertySymbols()可以获取指定对象的所有Symbol属性名，返回数组，成员是当前对象的所有Symbol属性名
Reflect.ownKeys可以返回所有类型的键名，包括常规键名和Symbol键名

Symbol.for() Symbol.keyFor()

Symbol.for()，先搜索已经登记的描述，找到相同的就返回，否则新建一个，登记到全局环境中
var s1 = Symbol.for('foo'); var s2 = Symbol.for('foo'); s1 === s2 // true
Symbol.keyFor()用来返回一个已经登记的Symbol的描述
var s1 = Symbol.for("foo"); Symbol.keyFor(s1) // "foo" var s2 = Symbol("foo"); Symbol.keyFor(s2) // undefined

Symbol.hasInstance()方法，当调用instanceof时，使用此方法
class m{ [Symbol.hasInstance](obj){ return obj instanceof Array } } [1,23,4] instanceof new m();
Symbol.isConcatSpreadable属性等于布尔值，规定用于concat方法时是否展开，数组默认行为可以展开,类似数组的对象默认不展开
let arr1 = ['c', 'd']; ['a', 'b'].concat(arr1, 'e') // ['a', 'b', 'c', 'd', 'e'] arr1[Symbol.isConcatSpreadable] // undefined let arr2 = ['c', 'd']; arr2[Symbol.isConcatSpreadable] = false; ['a', 'b'].concat(arr2, 'e') // ['a', 'b', ['c','d'], 'e']
Symbol.species指向当前对象的构造函数,定义方法如下
static get [Symbol.species]() { return this;//返回的内容为创建实例时使用的构造函数 }
Symbol.match指向一个函数，调用str.match(obj)时，会调用该属性
String.prototype.match(regexp) // 等同于 regexp[Symbol.match](this) class MyMatcher { [Symbol.match](string) { return 'hello world'.indexOf(string); } } 'e'.match(new MyMatcher()) // 1
Symbol.replace
Symbol.split
Symbol.search
Symbol.iterator
Symbol.toPrimitive
let obj = { [Symbol.toPrimitive](hint) { switch (hint) { case 'number': return 123; case 'string': return 'str'; case 'default': return 'default'; default: throw new Error(); } } }; 2 * obj // 246 3 + obj // '3default' obj == 'default' // true String(obj) // 'str'
Symbol.toStringTag
Symbol.unscopables,指向一个对象，该对象指定使用with关键字时，哪些属性会被排除，指定方法如下：
get [Symbol.unscopables]() { return { foo: true }; }

Set和Map的数据结构

Set成员值都是唯一的，set内部，判断相等用===，但是NaN是相等的
方法：
add() has() delete() clear()
用于遍历的方法 keys() values() entries() forEach(回调函数) 回调函数的参数依次为键值，键名，集合本身，forEach方法还可以有第二个参数，表示绑定的this对象。
由于set只有键值，所以 keys() values() 行为一致
Array.from可以将set转换成数组
实现并集，交集，差集
let a = new Set([1, 2, 3]); let b = new Set([4, 3, 2]); // 并集 let union = new Set([...a, ...b]); // Set {1, 2, 3, 4} // 交集 let intersect = new Set([...a].filter(x => b.has(x))); // set {2, 3} // 差集 let difference = new Set([...a].filter(x => !b.has(x))); // Set {1}
let set = new Set([1, 2, 3]); set = new Set([...set].map(val => val * 2)); // set的值是2, 4, 6 // 方法二 set = new Set(Array.from(set, val => val * 2));

WeakSet

成员只能是对象，引用后不计入垃圾回收机制，对象被回收释放内存不会考虑这里有没有引用
没有size属性，不可遍历，不支持clear
方法： add delete has

Map

各种类型的值都可以当做键名
方法set(键，值)（返回当前Map对象，因此可以写成链式） get(键)（不存在，显示undefined）clear() has() delete()（删除成功返回TRUE）
Map中可传入数组作为键值对，只要是具有Iterator接口的且每个成员都是一个双元素的数组的数据接口，都能转化成Map
严格相等的值视为同一个键，[]与[]不严格相等，{}与{}也不完全相等,
Set和Map中，NaN与NaN完全相等
size属性，返回长度
map[Symbol.iterator] === map.entries //TRUE
用于遍历的方法 keys() values() entries() forEach(回调函数) 回调函数的参数依次为键值，键名，集合本身，forEach方法还可以有第二个参数，表示绑定的this对象。
扩展运算符…可以将Map转为数组，转换成数组后，可以使用数组的map(),filter()方法

WeakMap

只接受对象作为键名，null除外，不接受null，键值正常
不计入垃圾回收机制
四个方法get()、set()、has()、delete()
没有size属性，不可遍历，不支持clear

Proxy

用于修改某些操作的默认行为，
let proxy = new proxy(target,handle)
生成一个实例，target表示要拦截的目标对象，handle的参数也是一个对象，用来定制拦截行为，要使一系列设置起作用，必须针对proxy实例进行操作
var handler = { get: function(target, name) { if (name === 'prototype') { return Object.prototype; } return 'Hello, ' + name; }, apply: function(target, thisBinding, args) { return args[0]; }, construct: function(target, args) { return {value: args[1]}; } }; var fproxy = new Proxy(function(x, y) { return x + y; }, handler); fproxy(1, 2) // 1 被apply拦截 new fproxy(1,2) // {value: 2} 被constructor拦截 fproxy.prototype === Object.prototype // true 被get拦截 fproxy.foo // "Hello, foo" 被get拦截
ownKeys方法返回的数组成员，只能是字符串或 Symbol 值。如果有其他类型的值，或者返回的根本不是数组，就会报错。
如果目标对象自身包含不可配置的属性，则该属性必须被ownKeys方法返回，否则报错。
另外，如果目标对象是不可扩展的（non-extensition），这时ownKeys方法返回的数组之中，必须包含原对象的所有属性，且不能包含多余的属性，否则报错。

14.Promise对象

异步编程的一种解决方案，是一个保存着未来才会结束的事件结果的容器
1.状态不受外界影响
2.一旦状态改变，就是最终状态 ,从pending 变为 fulfiled或者rejected,改变后就已经定型，成为resolved
Promise状态变为resolved，再抛出错误是无效的，在变为resolved前抛出错误有效
setTimeout是在下一轮事件中执行
done()用于回调链的尾端，捕获错误，不接受参数,但是需要手动在代码中写这个函数
Promise.prototype.done = function (onFulfilled, onRejected) { this.then(onFulfilled, onRejected) .catch(function (reason) { // 抛出一个全局错误 setTimeout(() => { throw reason }, 0); }); };
finally() 接受一个普通的回调函数作为参数

Node 有一个unhandledRejection事件，专门监听未捕获的reject错误。

process.on('unhandledRejection', function (err, p) { console.error(err.stack) });

上面代码中，unhandledRejection事件的监听函数有两个参数，第一个是错误对象，第二个是报错的 Promise 实例，它可以用来了解发生错误的环境信息。。

16.Generator

返回遍历器对象
for…of会自动调用遍历器对象的next()方法
解构赋值、扩展运算符、都是使用遍历器接口Iterator

一旦next方法的返回对象的done属性为true，for…of循环就会中止，且不包含该返回对象
Generator.prototype.throw()
generator的函数的catch语句执行一次就不执行了，throw函数接收一个参数，这个参数被catch语句接收，不要混淆generator的throw与全局的throw，全局的throw之后就不会在执行下文内容，generator的会
throw被抛出后，会附带执行一次next
Generator 函数体外抛出的错误，可以在函数体内捕获；反过来，Generator 函数体内抛出的错误，也可以被函数体外的catch捕获。但是一旦Generator执行过程中发生错误没被内部捕获，内部就不会执行下去了

只要 Generator 函数内部部署了try…catch代码块，那么遍历器的throw方法抛出的错误，不影响下一次遍历。
调用return ，如果有try…finally,会立刻执行finally里面内容一次，调用next，继续执行，finally执行完毕再调用next，执行return语句
调用return方法后，就开始执行finally代码块，然后等到finally代码块执行完，再执行return方法。

yield*

如果在 Generator 函数内部，调用另一个 Generator 函数，默认情况下是没有效果的。
